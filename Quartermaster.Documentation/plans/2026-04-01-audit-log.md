# Audit Log Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add audit logging for all state-changing operations across 6 entity types, with per-field change tracking and UI display on Member and Event detail pages.

**Architecture:** An `AuditLog` entity stores one row per field change (EntityType, EntityId, FieldName, OldValue, NewValue, Action, Timestamp, UserId). An `AuditLogRepository` provides write and query methods. Each data repository gets an `AuditLogRepository` dependency and logs changes. UserId is nullable — populated with "System" placeholder until auth exists (marked with TODOs).

**Tech Stack:** LinqToDB, FluentMigrator (M001), existing repository pattern

---

## Tasks

### Task 1: AuditLog entity, migration, repository, DTO

Create the data layer for audit logging.

**Files to create:**
- `Quartermaster.Data/AuditLog/AuditLog.cs` — entity
- `Quartermaster.Data/AuditLog/AuditLogRepository.cs` — data access
- `Quartermaster.Api/AuditLog/AuditLogDTO.cs` — DTO for API responses

**AuditLog entity fields:**
- `Guid Id` (PK)
- `string EntityType` (e.g., "Member", "Event", "Motion")
- `Guid EntityId`
- `string Action` (e.g., "Created", "Updated", "Deleted", "SoftDeleted")
- `string? FieldName` (null for Created/Deleted, set for field-level changes)
- `string? OldValue`
- `string? NewValue`
- `Guid? UserId` (nullable until auth exists)
- `string? UserDisplayName` (e.g., "System", later the actual username)
- `DateTime Timestamp`

**Table name constant:** `public const string TableName = "AuditLogs";`

**Add to M001 migration** (in Up(), before the indexes section):
```
Create.Table("AuditLogs")
    .WithColumn("Id").AsGuid().PrimaryKey()
    .WithColumn("EntityType").AsString(64)
    .WithColumn("EntityId").AsGuid()
    .WithColumn("Action").AsString(64)
    .WithColumn("FieldName").AsString(256).Nullable()
    .WithColumn("OldValue").AsCustom("TEXT").Nullable()
    .WithColumn("NewValue").AsCustom("TEXT").Nullable()
    .WithColumn("UserId").AsGuid().Nullable()
    .WithColumn("UserDisplayName").AsString(256).Nullable()
    .WithColumn("Timestamp").AsDateTime()
```

Add index: `Create.Index("IX_AuditLogs_EntityType_EntityId").OnTable("AuditLogs").OnColumn("EntityType").Ascending().OnColumn("EntityId").Ascending();`

Add to Down(): `DropTableIfExists("AuditLogs");`

**Add to DbContext:** `public ITable<AuditLog> AuditLogs => this.GetTable<AuditLog>();`

**Register** AuditLogRepository in `DbContext.AddRepositories()`.

**AuditLogRepository methods:**
- `Log(string entityType, Guid entityId, string action, string? fieldName, string? oldValue, string? newValue)` — creates single audit entry with UserId=null, UserDisplayName="System"
- `LogCreated(string entityType, Guid entityId)` — convenience for creation
- `LogFieldChange(string entityType, Guid entityId, string fieldName, string? oldValue, string? newValue)` — convenience for single field update
- `LogDeleted(string entityType, Guid entityId)` — convenience for deletion
- `GetForEntity(string entityType, Guid entityId)` — returns list ordered by Timestamp desc
